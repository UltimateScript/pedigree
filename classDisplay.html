Detailed Description

A display is either a dumb framebuffer or something more accelerated.

The general use case for writing to the screen:

  • Create a buffer with "newBuffer()".
  • Set the buffer as the main on-screen buffer with "setCurrentBuffer()".
  • Write to it, then update the changes to the screen with "updateBuffer()". Optionally here you can define a rectangle to be updated, usually referred to as a "dirty rectangle". * Optionally, you can use the faster acceleration functions "fillRectangle" and "bitBlit". Note that these functions may not actually be accelerated on-chip, this is up to the driver; however they are likely to be much faster than updating the buffer manually and calling updateBuffer().
